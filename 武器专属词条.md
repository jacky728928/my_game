# 武器专属词条设计文档

> 版本：v0.1 | 状态：**已实装**

---

## 一、专属词条总览

武器专属词条是副武器独有的升级选项，**只有在持有对应副武器时才可能在升级时被抽到**，与普通通用词条混合随机出现。

### 获取规则
1. 必须持有对应副武器（装备到槽位）
2. 升级时进入普通三选一/二选一选项池
3. 可无限叠加（每级递增）
4. 词条名使用对应武器的主题色

## 二、榴弹炮专属词条 — 火力倾泻 🔶

| 属性 | 值 |
|------|----|
| 词条 ID | `gl_firepower` |
| 所属武器 | 榴弹炮（`grenade_launcher`） |
| 主题色 | 橙色 `#e67e22` |
| 效果 | 每次发射的榴弹数量 **+1** |
| 叠加方式 | 线性递增（n级 = n颗额外榴弹） |
| 目标锁定 | 多颗榴弹各自锁定不同敌人 |
| 状态 | **已实装** |

**数量公式**：
```
实际发射数量 = 1 + 火力倾泻等级
```

| 等级 | 发射数量 | 说明 |
|------|---------|------|
| 0（初始） | 1 颗 | 锁定最近敌人 |
| 1 | 2 颗 | 锁定最近 + 第二近敌人 |
| 2 | 3 颗 | 锁定 3 个不同敌人 |
| 3 | 4 颗 | 锁定 4 个不同敌人 |
| n | 1+n 颗 | 敌人不足时复用最后目标 |

**触发流程**：
1. 冷却到期后，按弹体数量 `1+n` 搜索最近N个敌人
2. 每个目标都显示独立的目标锁定指示器
3. 多发榴弹从玩家位置飞向各自目标（各自飞行轨迹和爆炸）
4. 敌人数量不足时，剩余榴弹复用最后的目标位置

**伤害**：每颗榴弹独立计算（`基础攻击 × 800%`），数量越多总伤害越高。

## 三、手雷专属词条 — 集束手雷 🔴

| 属性 | 值 |
|------|----|
| 词条 ID | `gr_cluster` |
| 所属武器 | 手雷（`grenade`） |
| 主题色 | 红色 `#e74c3c` |
| 效果 | 手雷伤害 **+50% / 级** |
| 叠加方式 | 线性叠加（可无限升级） |
| 状态 | **已实装** |

**伤害公式**：
```
实际伤害倍率 = 15.0 × (1 + 0.5 × 集束手雷等级)
```

| 等级 | 伤害倍率 | 基础攻击=5时伤害 | 基础攻击=7时伤害 |
|------|---------|-----------------|-----------------|
| 0 | ×15.0 | **75** | **105** |
| 1 | ×22.5 | **112** | **157** |
| 2 | ×30.0 | **150** | **210** |
| 3 | ×37.5 | **187** | **262** |
| n | ×15.0(1+0.5n) | — | — |

**触发流程**：
1. 冷却到期后，搜索玩家周围 300px 内的敌人
2. 向最近敌人投掷手雷（正常飞行 + 引信 + 爆炸）
3. 爆炸伤害按上述公式计算（其他参数：AOE半径、引信时间不变）

## 四、医疗包专属词条 — 急救强化 💚

| 属性 | 值 |
|------|----|
| 词条 ID | `mk_healboost` |
| 所属武器 | 医疗包（`medkit`） |
| 主题色 | 绿色 `#2ecc71` |
| 效果 | 医疗包回复量 **+50% / 级** |
| 叠加方式 | 线性叠加（可无限升级） |
| 状态 | **已实装** |

**回复公式**：
```
实际回复倍率 = 20.0 × (1 + 0.5 × 急救强化等级)
```

| 等级 | 回复倍率 | 基础攻击=5时回复 | 基础攻击=7时回复 |
|------|---------|-----------------|-----------------|
| 0 | ×20.0 | **+100 HP** | **+140 HP** |
| 1 | ×30.0 | **+150 HP** | **+210 HP** |
| 2 | ×40.0 | **+200 HP** | **+280 HP** |
| 3 | ×50.0 | **+250 HP** | **+350 HP** |
| n | ×20.0(1+0.5n) | — | — |

**触发流程**：
1. 冷却到期后自动触发（无需目标）
2. 按当前等级计算回复值
3. `player.hp += heal`，不超过 `player.maxHp`
4. 玩家头顶弹出绿色 "+数值" 治疗数字

## 五、UI 表现

### 升级卡片
- 卡片顶部显示 **`[武器名 专属]`** 小标签（武器主题色）
- 卡片边框和名称文字使用武器主题色
- 悬停时同样显示高亮背景

### 通用词条 vs 专属词条区分

| 元素 | 通用词条 | 专属词条 |
|------|---------|---------|
| 标签 | 无 | `[武器名 专属]`（武器色） |
| 名称 | 金色 `#f1c40f` | 武器主题色 |
| 边框 | 金色 `#f1c40f` | 武器主题色 |
| 持有要求 | 无 | 必须持有对应副武器 |

## 六、代码映射（便于查阅）

| 武器 | 词条名 | 数据位置 |
|------|-------|---------|
| 榴弹炮 | 火力倾泻 | `SECONDARY_ABILITY_POOL.grenade_launcher` → `player.secondaryAbilityLevels['gl_firepower']` |
| 手雷 | 集束手雷 | `SECONDARY_ABILITY_POOL.grenade` → `player.secondaryAbilityLevels['gr_cluster']` |
| 医疗包 | 急救强化 | `SECONDARY_ABILITY_POOL.medkit` → `player.secondaryAbilityLevels['mk_healboost']` |

**核心触发点**：
- 词条抽取：`js/ui.js → pickRandomChoices()`（从 `ABILITY_POOL` 通用词条 + 玩家已装备副武器对应的 `SECONDARY_ABILITY_POOL` 词条混合抽取）
- 词条选择：`js/player.js → Player.applyAbility()`（按 `abilityId` 累计等级，副武器词条走 default 分支累加）
- 武器发射计算：`js/combat.js → updateSecondaryWeapons()`（在 heal / aoe_direct / aoe_delayed 分支中读取对应 `secondaryAbilityLevels` 值，动态计算治疗量 / 弹数 / 伤害倍率）
